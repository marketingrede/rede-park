**Problema:** Employee, Vehicle, ContactAttempt, SeniorImport não definem relationships. Força queries separadas + mapeamento manual em JS.

| Model | File | Relationship |
|-------|------|-------------|
| `Employee` | `app/models/employee.ts` | `@hasMany(() => Vehicle)` |
| `Vehicle` | `app/models/vehicle.ts` | `@belongsTo(() => Employee)` |
| `ContactAttempt` | `app/models/contact_attempt.ts` | `@belongsTo(() => Employee)`, `@belongsTo(() => Visitor)` |

**Referência:** `app/models/approval_request.ts` já usa `@belongsTo(() => Employee)` corretamente.

### A4. Bulk destroy batch + transaction

**`app/controllers/employees_controller.ts:92-110`:**
```typescript
// ANTES (N+1):
for (const id of ids) {
  const employee = await Employee.find(id)
  if (employee) await employee.delete()
}

// DEPOIS (batch):
await Employee.query().whereIn('id', ids).delete()
```

**`app/controllers/imports_controller.ts:95-126`:**
- Envolver em `db.transaction()`
- Deletar employees em batch: `Employee.query().whereIn('senior_import_id', ids).delete()`
- Deletar imports em batch: `SeniorImport.query().whereIn('id', ids).delete()`
- Unlink arquivos em loop (não dá pra batch, I/O)
